# Техническое задание на разработку информационной системы для интернет-магазина

# 1. Изучение предметной области

# 1.1. Основные бизнес-процессы интернет-магазина

Прием заказов от клиентов – осуществляется менеджером по продажам. Фиксируется заказ, клиент, список товаров, количество, цена, а также статус обработки.

Поступление товаров от поставщиков – выполняется кладовщиком. Оформляется приходная накладная с указанием склада, товаров, количества и закупочных цен.

Реализация товаров – осуществляется на основании заказов или напрямую. Отгрузка фиксируется документом реализации. Склад списывает товары, происходит продажа.

Учет остатков – система постоянно отслеживает количество товаров на каждом складе. Автоматически обновляется при поступлениях и реализациях.


# 1.2. Документы, используемые в учете

Заказ покупателя – номер, дата, клиент, менеджер, табличная часть (товары, количество, цена), статус.

Поступление товаров – номер, дата, склад, табличная часть (товары, количество, закупочная цена).

Реализация товаров – номер, дата, клиент, склад, табличная часть (товары, количество, розничная цена).


# 1.3. Пользователи и их роли

Администратор – имеет полный доступ, управляет системой и пользователями.

Менеджер по продажам – работает с клиентами и заказами, оформляет отгрузки.

Кладовщик – регистрирует поступления, контролирует остатки.


# 2. Сбор и структурирование требований

# 2.1. Функциональные требования

Ведение справочников: Номенклатура, Клиенты, Склады, Менеджеры.

Регистрация документов: Заказ покупателя, Поступление товаров, Реализация товаров.

Контроль остатков при оформлении заказов и реализаций.

Автоматическое списание товаров со склада при реализации.

Поддержка статусов заказов.

Формирование отчетов:

Остатки товаров на складах

Продажи за период

Движение товаров


Возможность поиска, фильтрации данных.

Проведение документов с проверкой корректности.

Печать документа «Заказ покупателя» и «Реализация товаров».


# 2.2. Нефункциональные требования

Платформа: 1С:Предприятие 8.3, последняя стабильная версия.

Производительность: до 500 товаров, 1000 клиентов, 5000 заказов в год.

Интерфейс: удобный, типовой 1С-интерфейс с панелью навигации.

Безопасность: разграничение прав доступа по ролям.

Соответствие стандартам: код соответствует рекомендациям фирмы 1С.


# 2.3. Приоритеты требований

Критичные: учет заказов, реализация, остатки, справочники, доступ по ролям.

Средний приоритет: отчеты, печатные формы.

Дополнительные: фильтры, поиск, интерфейсная доработка.


# 3. Основные объекты системы

# 3.1. Справочники

Название Назначение Основные реквизиты

Номенклатура Учет товаров Артикул, наименование, единица измерения, цены
Клиенты Учет покупателей ФИО/наименование, телефон, email, адрес доставки
Склады Учет мест хранения товаров Наименование склада
Менеджеры Учет сотрудников ФИО, должность


# 3.2. Документы

Документ Назначение Структура данных

Заказ покупателя Регистрация заказа Дата, номер, клиент, менеджер, товары, статус
Поступление товаров Учет прихода товаров Дата, номер, склад, товары, закупочная цена
Реализация товаров Учет отгрузки и списания товаров Дата, номер, клиент, склад, товары, цена


# 3.3. Регистры накопления

ОстаткиТоваров – фиксирует количество товара на складе.

Продажи – фиксирует факты реализации, продажи по товарам и клиентам.


# 4. Проектирование структуры данных

# 4.1. Реквизиты справочников и документов

Примеры:

Номенклатура: строка (артикул), строка (наименование), число (цена), справочник (единица измерения).

Документы: дата (дата оформления), строка (номер), справочники (ссылки на клиента, менеджера, склад).


# 4.2. Табличные части документов

Включают список товаров: товар (ссылка), количество (число), цена (число).

Общая сумма может рассчитываться автоматически.


# 5. Бизнес-процессы и сценарии использования

# 5.1. Жизненный цикл документов

Поступление товаров:

1. Кладовщик оформляет поступление.


2. Товары добавляются в остатки на складе.



Оформление заказа:

1. Менеджер создает заказ.


2. Проверяется наличие товара.


3. Заказу присваивается статус.



Реализация товара:

1. Менеджер формирует реализацию (на основании заказа или отдельно).


2. Товары списываются со склада.


3. Документ можно распечатать (накладная, чек).




# 5.2. Правила проверки данных

Нельзя провести заказ, если товаров нет в наличии.

При списании проверяется остаток на складе.

Проверка обязательных полей (дата, клиент, товар и т.д.)


# 5.3. Пользовательские сценарии

Менеджер: оформляет заказы, формирует реализацию, работает со справочниками.

Кладовщик: регистрирует поступления, проверяет остатки.

Администратор: управляет пользователями, правами и настройками системы.


# 6. Роли пользователей и права доступа

Роль Доступ к объектам

Администратор Полные права на все справочники, документы, регистры и настройки
Менеджер по продажам Просмотр справочников, создание/редактирование заказов и реализаций
Кладовщик Создание поступлений, просмотр остатков, без доступа к заказам
